# Product Schema

## Required Fields
- sku
- name
- category
- status
- price
- short_description
- target_customer
- key_benefits
- specifications
- warranty
- faq
- troubleshooting
- cross_sell
- upsell
- seo_keywords
- last_updated

## Recommended Format
Use Markdown plus YAML front matter for each product.

Example:

```md
---
sku: HUSSAR1000RS
name: Hussar 1000RS
category: Curved Treadmill
status: active
price: 0
---

# Hussar 1000RS

## Overview

## Target Customer

## Key Benefits

## Specifications

## Warranty

## FAQ

## Troubleshooting

## Cross Sell

## Upsell
```

## Purpose
This schema keeps every product consistent and machine-readable.
